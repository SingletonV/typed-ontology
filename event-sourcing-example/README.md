# Пример схемы событий для студенческой базы данных

- Создание каждой сущности: студент, курс
- Модификация атрибутов
- Запись на курс
- Отчисление с курса
- Увеличение целочисленного атрибута (например, credits)
- Добавление элементов в список

По каждому событию:
- userId
- timestamp

Запросы:
- Текущий состав атрибутов сущности
- Текущий состав атрибутов сущности на момент времени
- Сжатие цепочки событий
- Blame (кто, когда поменял заданный атрибут)
